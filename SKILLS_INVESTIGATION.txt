HOW MATCHING WORKS WITHOUT SKILLS
----------------------------------
The system does NOT rely on skills alone. It uses hybrid search (dense + sparse):

1. DENSE VECTOR (semantic embedding)
   - Profile text (headline, description, roles, companies) is embedded
   - Query text is embedded and compared via cosine similarity
   - Finds candidates even without explicit skills if their profile text is relevant
   - Example: "Python developer" matches a profile with headline "Sr. Python Dev"

2. SPARSE VECTOR (BM25 keyword matching)
   - Exact keyword overlap scoring
   - Limited for skill-less profiles but still catches title/headline matches

3. RANKING BONUSES (post-retrieval scoring):
   - Skills perfect match:   +0.40
   - Skills partial match:   +0.25
   - Title exact match:      +0.20
   - Company exact match:    +0.25
   - Location city exact:    +0.15
   - Education match:        +0.12
   - Experience fit:         +0.12
   - Certification match:    +0.06
   - Multi-signal combo:     +0.15

4. SKILL EXPANSION (relationship graph)
   - "Python" auto-expands to Django, Flask, FastAPI, Pandas, etc.
   - Finds candidates who list related skills even if exact skill missing

PROFILES WITHOUT SKILLS CAN STILL BE FOUND IF:
  - Their headline/title contains relevant keywords (title matching + embedding)
  - Their work history/description mentions relevant experience (embedding)
  - Their location/company/education matches (bonus scoring)

PROFILES WITHOUT SKILLS WILL RANK LOWER because they miss the +0.40 skills bonus,
but they are NOT invisible.
